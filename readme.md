# Library of Deception

A small archive site that browses a folder of documents the way a GitHub
repository browses its files: a clean table of everything on the shelf,
with a slide-over reader and one-click downloads. No accounts, no
branches, no commits — just the files, laid out and readable.

## How it fits together

```
readme.md
output.py            <- reads docs/, writes data/files.json
server.js            <- Express app: serves web/, exposes the file API
package.json
docs/                <- the actual archive: drop .txt (or similar) files here
data/
  files.json         <- generated by output.py, do not hand-edit
web/
  index.html
  css/
    style.css
  scripts/
    main.js
```

**output.py** is the only thing that reads `docs/`. It walks the folder,
reads each file, tidies up whitespace, and writes one JSON index to
`data/files.json` containing every file's name, size, line count,
last-modified time, a one-line preview, and its full content.

**server.js** never touches `docs/` directly. On startup (and whenever
`/api/reindex` is called) it runs `output.py`, then serves
`data/files.json` through a small API, and serves `web/` as static files.

**web/** is a plain HTML/CSS/JS frontend. `scripts/main.js` calls the API,
renders the file table, and opens a reader panel when a row is clicked.
`css/style.css` holds the whole visual design: black-and-white, monospace
accents, and a soft "decode" animation on file names.

## Running it

Needs Node.js and Python 3, nothing else — `server.js` is plain Node
(no npm packages to install) and `output.py` is plain Python.

```bash
npm start
# or: node server.js
# -> Library of Deception is open at http://localhost:3000
```

Add or edit files in `docs/`, then either restart the server or click the
**reindex** button in the top of the file table — both re-run `output.py`
and refresh what the page shows.

## API

| Route                     | Method | Returns                                   |
|---------------------------|--------|--------------------------------------------|
| `/api/repo`                | GET    | Repo metadata + every file's full record   |
| `/api/files/:name`         | GET    | A single file's record                     |
| `/api/download/:name`      | GET    | The raw file, as a download                |
| `/api/reindex`             | POST   | Re-runs `output.py`, returns the fresh index |

## Adding documents

Drop any text file into `docs/`. The next reindex picks it up
automatically — no code changes needed. `output.py` currently labels a
few common extensions (`.txt`, `.md`, `.json`, `.py`, `.js`, `.csv`,
`.log`); anything else is still indexed, just labeled `TEXT`.
